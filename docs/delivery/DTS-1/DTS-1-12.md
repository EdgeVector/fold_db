# DTS-1-12 Fix Complex Chain Expression Execution

## Description

Fix execution engine issues with complex chain expressions like `blogpost.map().content.split_by_word().map()` that are failing in unit tests. The execution engine is not correctly handling the relationship between chain depth and emission depth, causing `evaluate_field_expression` to look for contexts at depths that don't exist.

## Requirements

### Problem Statement
The test `test_complex_chain_expression` in `tests/unit/transform/chain_parser_integration_tests.rs` is failing because:

1. **Depth Mismatch**: The execution engine is trying to iterate to depth 2 (emission depth), but the iterator stack only has contexts at depths 0 and 1
2. **Context Resolution**: The `evaluate_field_expression` function is looking for a context at depth 2 that doesn't exist
3. **Expected vs Actual**: The test expects a single string result, but the execution engine produces an array of 4 null values

### Technical Details
- **Failing Test**: `unit::transform::chain_parser_integration_tests::test_complex_chain_expression`
- **Expression**: `blogpost.map().content.split_by_word().map()` (depth: 2)
- **Expected Result**: `"Complex chain parsed content"` (single string)
- **Actual Result**: `[Null, Null, Null, Null]` (array of 4 null values)

### Root Cause Analysis
1. **Iterator Stack Management**: The execution engine creates contexts at depths 0 and 1, but the chain has depth 2
2. **Iteration Logic**: The `execute_one_to_one` function calls `iterate_to_depth` with the emission depth (2), but should iterate to the appropriate depth for the chain
3. **Field Expression Evaluation**: The `evaluate_field_expression` function is not using the correct current item from the iterator stack

## Implementation Plan

   - Modify `execute_one_to_one` to iterate to the correct depth

## Notes

This task addresses a fundamental architectural issue in the execution engine where complex chains with multiple operations are not being handled correctly. The fix requires careful analysis of the relationship between chain depth, emission depth, and iterator stack context management.

The issue is particularly complex because it involves the interaction between:
1. Chain parsing (determining depth)
2. Iterator stack management (creating contexts)
3. Execution engine logic (iterating and evaluating)
4. Field expression evaluation (resolving values)

Success in this task will significantly improve the robustness of the declarative transform execution system.
